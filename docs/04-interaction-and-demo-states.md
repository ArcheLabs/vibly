# Vibly Frontend Preview - Interaction and Demo States

## 1. 文档目的

本文档定义前端预览版必须具备的交互行为与演示状态。

因为本轮不接入后端，因此所有交互都允许用本地 UI state 和 mock data 实现。但它们仍必须具备真实产品的基本行为逻辑，不能只是静态截图拼接。

---

## 2. 全局交互原则

1. 所有一级页面都必须可切换
2. 关键入口点击后必须有可见反馈
3. 预览版中的行为应尽量模拟真实产品路径
4. 所有“不可用”区域必须有明确说明
5. 不允许出现用户点击后无反馈的关键按钮

---

## 3. 一级导航交互

### 3.1 切换页面

用户点击左侧导航栏后：
- 更新当前一级页面
- 左侧列表区切换为对应页面内容
- 右侧交互区切换为对应页面内容

### 3.2 保留上次状态

建议：
- 每个一级页面保留最近一次选中项
- 再次切回时恢复该状态

若实现复杂，可先只实现基础切换。

---

## 4. 聊天页交互

# 4.1 点击会话列表项

结果：
- 设置当前会话为选中态
- 右侧显示对应聊天头部、消息记录、输入区
- 会话项高亮
- 未读数可清零（仅本地演示）

---

# 4.2 未选中会话状态

触发方式：
- 初始无默认选中
- 或通过“返回空状态”按钮进入

结果：
- 右侧显示聊天空白占位页

---

# 4.3 打开身份切换弹窗

触发方式：
- 点击聊天头部右侧身份切换区域

结果：
- 弹出 IdentitySwitcher
- 显示当前用户真人身份 + 我的智能体列表

---

# 4.4 切换当前身份

触发方式：
- 在 IdentitySwitcher 中点击某个身份

结果：
- 更新当前身份
- 关闭弹窗
- 聊天头部当前身份显示更新
- 输入区底部提示更新
- 新发送消息使用新身份

---

# 4.5 点击当前已选身份

结果：
- 仅关闭弹窗
- 不发生其他变化

---

# 4.6 本地发送消息

触发方式：
- 输入文本
- 点击发送按钮

结果：
- 将消息追加到当前会话消息列表
- 发送者为当前用户选中身份
- 清空输入框
- 滚动到消息底部（可选）

说明：
这仅为 UI 演示，不需要模拟服务端返回。

---

# 4.7 点击真人头像

触发方式：
- 在聊天头部点击真人头像
- 在消息头像中点击真人头像（可选）
- 在联系人页点击真人

结果：
- 打开用户信息页

---

# 4.8 点击智能体头像

触发方式：
- 在聊天头部点击智能体头像
- 在发现页点击智能体详情
- 在联系人页点击智能体

结果：
- 打开智能体信息页

---

## 11. 演示建议场景脚本

为了便于演示，可以在 mock 数据中预置以下场景。

### 场景 1：普通真人会话
- 与真人聊天
- 可以正常发送

### 场景 2：与收费智能体会话
- 头部有收费提示
- 消息区出现限制卡片

### 场景 3：与停用智能体会话
- 输入区不可用
- 有停用提示

### 场景 4：从发现页进入聊天
- 体现产品路径

### 场景 5：切换为自己的智能体发言
- 体现双身份逻辑

---

## 12. 对 Codex 的交互约束

Codex 在实现交互时必须遵守：

1. 仅使用本地 UI state
2. 不创建伪 API 层
3. 不引入复杂状态管理库
4. 所有按钮都应有明确反馈或说明
5. 所有状态都应通过 mock 数据或本地切换进入
